- **여백**: 충분히 확보 (답답하지 않게)

- **아이콘**: 이모지 또는 심플한 아이콘 사용

## 📱 촬영 방법

### 방법 1: 실제 화면 캡처 (권장)

1. **앱 실행**
   ```bash
   streamlit run app.py
   ```

2. **브라우저를 모바일 뷰로 변경**
   - Chrome 개발자 도구 (F12)
   - Toggle device toolbar (Ctrl+Shift+M)
   - 디바이스 선택: iPhone 14 Pro Max
   - 해상도 확인: 1290 x 2796

3. **화면 캡처**
   - macOS: Cmd+Shift+4, 스페이스바, 클릭
   - Windows: Snipping Tool 또는 Win+Shift+S
   - Chrome: 개발자 도구 → ... → Capture screenshot

4. **해상도 조정** (필요시)
   - Photoshop, GIMP, 또는 온라인 도구 사용
   - 정확히 요구되는 해상도로 크롭/리사이즈

### 방법 2: 목업 사용 (더 예쁨)

**도구**:
- [Mockuphone](https://mockuphone.com/)
- [Screely](https://www.screely.com/)
- [Smartmockups](https://smartmockups.com/)
- [Figma](https://www.figma.com/) (무료)

**과정**:
1. 화면 캡처 (배경 제거)
2. 목업 도구에 업로드
3. 디바이스 프레임 선택
4. 텍스트 오버레이 추가
5. 다운로드

### 방법 3: 전문 디자인 도구

**Adobe XD / Figma**:
1. 캔버스 크기를 요구 해상도로 설정
2. 앱 화면 스크린샷 배치
3. 텍스트, 아이콘, 배경 추가
4. 내보내기 (PNG, 100% 품질)

## 📝 스크린샷 체크리스트

### 기술 요구사항
- [ ] 정확한 해상도 (1290x2796 for iPhone 6.7")
- [ ] PNG 또는 JPG 형식
- [ ] 파일 크기 < 8MB (각 스크린샷)
- [ ] 상태바 없음 (또는 깨끗하게 편집)
- [ ] 날짜/시간은 일관되게 (예: 모두 같은 날짜)

### 내용 요구사항
- [ ] 개인정보 없음 (이메일, 전화번호 등)
- [ ] 더미 데이터 사용 (실제 학교명 대신 "OO 초등학교")
- [ ] 오타 없음
- [ ] 텍스트 읽기 쉬움
- [ ] 주요 기능이 명확하게 보임

### 디자인 요구사항
- [ ] 일관된 색상 팔레트
- [ ] 깔끔한 레이아웃
- [ ] 충분한 여백
- [ ] 고품질 (흐릿하지 않음)
- [ ] 텍스트 오버레이가 UI를 가리지 않음

## 💡 팁

### 1. 일관성 유지
모든 스크린샷에서 동일한:
- 시간 (예: 9:41)
- 배터리 레벨
- 신호 강도
- 날짜 형식

### 2. 밝고 명확하게
- 밝은 배경 사용
- 충분한 대비
- 텍스트는 크고 읽기 쉽게

### 3. 실제 사용 사례
- 더미 데이터는 현실적으로
- "테스트", "샘플" 같은 단어 피하기
- 실제로 사용할 법한 데이터

### 4. 로컬라이제이션
- 한국어 스크린샷 세트
- 영어 스크린샷 세트 (글로벌 출시 시)

### 5. 주기적 업데이트
- 앱 UI가 변경되면 스크린샷도 업데이트
- 새 기능 추가 시 새 스크린샷 추가
